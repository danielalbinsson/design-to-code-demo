# Sync Pipeline

**Mandatory:** Follow this skill end-to-end whenever you touch design tokens or anything that consumes them. Do not finish the task until the pipeline steps below are done.

## Source of truth

| Layer | Path | Never edit |
|-------|------|------------|
| DTCG JSON (source) | `packages/tokens/` | — |
| Style Dictionary | `packages/acme-tokens/config.mjs` | — |
| Generated CSS | `packages/acme-tokens/build/*.css` | Hand-edit |
| App token CSS (copy) | `packages/acme-app/src/tokens/*.css` | Hand-edit |

Figma variables are **visual reference** — JSON is the build source of truth. See [figma.md](../../figma.md).

## Token edit map

| Change type | Edit here |
|-------------|-----------|
| Shared spacing, radius, type sizes | `packages/tokens/primitive/` |
| Brand primitives | `packages/tokens/brand-alpha/`, `brand-beta/` |
| Semantic aliases | `packages/tokens/semantic/` |
| Shared dark overrides | `packages/tokens/dark/semantic/` |
| Brand dark overrides | `packages/tokens/dark/brand-alpha/`, `dark/brand-beta/` |
| New Tailwind utility | `packages/acme-app/src/index.css` (`@theme inline`) |
| New brand / theme file | `packages/acme-tokens/config.mjs` + `ThemeProvider` + Storybook preview |

**Dark mode rule:** `accent-soft` and `bg-surface-alt` must differ from `bg-surface` in every brand dark mode. Define brand-specific dark accents in `dark/brand-*/`, not shared gray aliases that collapse to surface.

## Workflow (run in order)

Copy and track:

```
Sync pipeline:
- [ ] 1. Edit DTCG JSON (or config) only — no hand-edited CSS
- [ ] 2. Rebuild and copy tokens
- [ ] 3. Inspect generated CSS for all 4 themes
- [ ] 4. Wire new semantics into app (if any)
- [ ] 5. Remove component workarounds — use tokens only
- [ ] 6. Update docs
- [ ] 7. Verify build + Storybook
- [ ] 8. Figma reconciliation (if user wants design parity)
```

### 1–2. Rebuild

Always run after token JSON changes:

```bash
CI=true corepack pnpm build:tokens && CI=true corepack pnpm build:tokens:json && CI=true corepack pnpm copy:tokens
```

Or `CI=true corepack pnpm build` for full app compile.

### 3. Inspect generated CSS

Grep all four outputs in `packages/acme-tokens/build/`:

- `brand-alpha.css`, `brand-alpha.dark.css`, `brand-beta.css`, `brand-beta.dark.css`

Confirm changed semantics resolved correctly. For color tokens, verify contrast pairs:

- `accent-soft` vs `bg-surface` and `bg-page`
- `action-primary` vs `bg-surface`
- `bg-surface-alt` vs `bg-surface`
- `border-default` vs `bg-surface`

Flag collisions (same resolved value on foreground/background pairs used in UI).

### 4. Wire new semantics

If you added a **new semantic token**:

1. Map it in `packages/acme-app/src/index.css` under `@theme inline` if components need a Tailwind utility.
2. Document in `docs/tokens.md`.
3. Update affected `docs/components/*.md` if component usage changes.

If you added a **new brand**: follow [docs/theming.md](../../docs/theming.md).

### 5. Components use semantics only

- No hardcoded hex in components or dashboard patterns.
- No `useTheme()` branches to pick colors — fix tokens instead.
- Reference utilities like `bg-action-primary`, `bg-accent-soft`, `text-text-secondary`, not `--color-blue-500`.

### 6. Update docs

| Artifact | When |
|----------|------|
| `docs/tokens.md` | Architecture, new semantics, dark rules |
| `docs/components/*.md` | Component token usage changed |
| `tokens.dtcg.json` | Regenerated by `build:tokens:json` — commit if repo tracks it |
| `pnpm build:llms` | Run when component/token docs changed for AI agents |

```bash
CI=true corepack pnpm build:llms
```

### 7. Verify

```bash
CI=true corepack pnpm build
CI=true corepack pnpm build-storybook
```

Manually spot-check all four combinations in the app or Storybook toolbar: Alpha/Beta × light/dark. Pay attention to stat cards, charts, table headers, and primary buttons.

### 8. Figma (optional, on request)

Code does not auto-push to Figma. When the user wants design parity:

1. Load `figma-generate-library` + `figma-use` for variable/component reconciliation.
2. Load `da-update-figma` for surgical screen updates — never wipe pages.

Figma file: [Acme Design System](https://www.figma.com/design/mujPrD4AdCQhg3uRwFcNJK/) (`mujPrD4AdCQhg3uRwFcNJK`). Dark modes: `Brand Alpha · Dark`, `Brand Beta · Dark`.

## Anti-patterns

- Hand-editing `packages/acme-app/src/tokens/*.css`
- Theme-specific color logic in React when a token fix suffices
- Setting `accent-soft` to the same value as `bg-surface` in dark mode
- Finishing a token task without running rebuild + build verify
